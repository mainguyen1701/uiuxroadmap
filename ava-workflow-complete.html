# Ava Workflow: Linear-Style AI-Assisted Admissions Management *Updated with Best UI/UX Practices · March 2026* **Based on:** Symphony agent pattern + The Ava Handbook principles **Author:** Emma Nguyen **Design:** Hoang Nguyen --- ## 1. Overview **Ava** is Applicaa's AI agent for Admissions+. She operates inside the platform on behalf of admissions staff — triaging applications, detecting duplicates, drafting communications, and updating records — with a human always in the loop for review and approval. ### KEY PRINCIPLE > The admissions coordinator's only required actions are: **(1)** describe the task, and **(2)** approve or redirect the outcome. Ava handles everything in between. ### Core Principles from The Ava Handbook: - **Labour Visibility:** Every action visible with real-time progress - **Human-in-the-Loop:** L2 (Draft & Confirm) default for irreversible actions - **Confidence Transparency:** Show scores (+ reasoning) on every suggestion - **Brand Integration:** "Ava by Applicaa" co-branded experience - **Safeguarding:** Automatic escalation for sensitive cases --- ## 2. Error Handling & Edge Cases ### 4.1 Low Confidence (< 70%) ``` ┌─ Low Confidence — Human Review Required ────────────────┐ │ │ │ ⚠️ Confidence: 55% — Below threshold for auto-action │ │ │ │ Ava's recommendation: │ │ "I'm uncertain about this batch. The records show │ │ unusual patterns that don't match typical cases." │ │ │ │ Why: │ │ • Mixed engagement signals │ │ • Unusual application timing │ │ • Limited historical data for this intake │ │ │ │ [Escalate to Senior Staff] [Proceed Anyway] │ │ [Request More Analysis] [Cancel Task] │ │ │ └───────────────────────────────────────────────────────────┘ ``` **Action:** Always escalate to human when confidence < 70% ### 4.2 Analysis Failure ``` ┌─ Analysis Failed ──────────────────────────────────────┐ │ │ │ ❌ Unable to complete analysis │ │ │ │ Error: Database connection timeout │ │ │ │ Ava has attempted 3 times. │ │ │ │ [Retry Analysis] [Escalate to IT] [Cancel Task] │ │ │ └───────────────────────────────────────────────────────────┘ ``` **Auto-retry:** 3 attempts with exponential backoff **Fallback:** Escalate to human/IT ### 4.3 User Rejection with Feedback ``` Coordinator: "These students are all from the same family — don't chase them together" ↓ Ava: "Understood! I'll group by family and stagger the chases." ↓ [Back to Step 4 with new parameters] ``` **Loop:** Request Changes → Back to Step 4 → Re-analyse → New plan --- ## 5.
