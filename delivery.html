Review Cadence
Five named checkpoints with go / no-go framing. Each gate must pass before the next phase begins or the next build starts.
1
Phase 0 → Phase 1 gate  Feb 2026 ✓
DS Lookbook peer-reviewed by engineering. All components, tokens, and patterns published and agreed. Emma signs off Phase 0 complete.
Go condition: Lookbook live at designsaplus.com/lookbook with complete component set. Engineering confirms all tokens are implementable. No-go: Missing token categories or unreviewed components.
2
Phase 1 item-level gates  Mar 2026 ◉
Each of the 8 Phase 1 items signs off independently. PM sign-off per item before engineering build. Each item is done when it is live in the product.
Go condition: Prototype reviewed and approved by PM. Engineering confirms no blocking dependencies. No-go: Prototype not user-tested OR engineering dependencies unresolved (items 1.1, 1.5).
3
Phase 2 prototype sign-off  Apr 2026
Each prototype tested with a minimum of 5 school staff. PM signs off after user testing. Engineering must not start builds until prototype passes the 80% task success threshold.
Go condition: ≥5 school staff tested. ≥80% task success on core flows. PM sign-off received. No-go: Below threshold task success OR fewer than 5 testers OR PM sign-off not given.
4
Phase 3 deliverable sign-off  Mid-Apr 2026
DS Standards Report and Governance Framework must be signed off by all product leads. Linter config running in CI before Phase 3 is closed.
Go condition: All product leads have reviewed and approved. Linter live in CI. Governance Framework published in DS Lookbook. No-go: Any product lead has not reviewed OR linter not running.
5
Phase 4 per-stage gates  Jun / Sep / Dec 2026
Each AI stage user-tested with ≥3 school staff before release. Human override interface requires legal/safeguarding sign-off before Stage 1 build begins (deadline: May 20).
Go condition: ≥3 school staff tested per stage. Legal/safeguarding sign-off on override interface (Stage 1). AI DS components in Lookbook before first Stage ships. No-go: Legal sign-off pending OR insufficient user testing.
